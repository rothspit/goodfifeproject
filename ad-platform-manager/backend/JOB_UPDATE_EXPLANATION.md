# ジョブ更新システムについて

## 重要な発見 🎉

調査の結果、**ジョブ更新とヘブン更新は同じボタン**を使用することが判明しました！

### システム詳細

- **更新ボタン**: どちらも「カウンターの更新」ボタン
- **残り回数**: 共通カウンター（15/16回など）
- **場所の違い**:
  - **ヘブン更新**: ダッシュボード → MENU → 「カウンターの更新」
  - **ジョブ更新**: ダッシュボード → 「求人管理」タブ → 「カウンターの更新」

### 実装方針

既存のヘブン更新システムが1日15回実行するため、**ジョブ更新専用のスケジューラーは不要**です。

#### オプション1: 既存システムを活用（推奨）
ヘブン更新が自動で15回実行するので、ジョブ更新も自動で完了しています。

#### オプション2: ジョブ更新専用スクリプト（手動実行用）
1日1回、「求人管理」タブから更新を実行するスクリプトを作成可能。

### 推奨事項

**追加の自動化は不要です**。理由:
1. ヘブン更新が1日15回実行中
2. 同じカウンターを共有しているため、ジョブ更新も自動で消化される
3. 残り回数は15/16回から徐々に減少

### もし手動でジョブ更新したい場合

```bash
cd /home/user/webapp/ad-platform-manager/backend
npx ts-node job-update-manual.ts
```

このスクリプトを作成すれば、「求人管理」タブから手動で更新できます。

## 結論

**現状のヘブン更新スケジューラーで十分**です。ジョブ更新も自動で実行されています。
